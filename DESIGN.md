# QQ 音乐 Loon 激进清理插件设计

日期：2026-07-11

## 目标

为 iOS 版 QQ 音乐构建一个激进模式的 Loon 插件，用来减少或移除开屏广告、启动弹窗、运营卡片，以及直播、短视频、商城、活动、抽奖等不想要的 tab 或入口。

用户已选择激进模式，并接受对 QQ 音乐相关域名启用 MITM。因此插件以响应体 JSON 清洗为主，以 URL rewrite 拦截为辅。

## 非目标

- 不屏蔽核心听歌功能：登录、搜索、播放、歌单、专辑、歌手、歌词、收藏、下载、缓存、会员状态。
- 不对所有腾讯域名做宽泛 MITM。
- 不承诺一次性永久适配 QQ 音乐未来所有接口变化。插件需要便于根据 Loon 请求日志继续微调。
- 不创建额外 UI 或管理应用。

## 产物

- `loon/QQMusic.AggressiveCleaner.plugin`：Loon 插件主文件。
- `loon/scripts/qqmusic_cleaner.js`：响应体清洗脚本。
- 本设计文档。

## 架构

插件分为四层。

### 1. Rewrite 层

插件使用 `[Rewrite]` 规则处理明确的广告、开屏、运营和上报接口。对于确认不属于核心功能的 URL，使用 `reject`、`reject-dict` 或 Loon 支持的等效 rewrite 动作。

Rewrite 规则按注释分组：

- 开屏和启动广告。
- 运营、活动和弹窗配置。
- 曝光、点击和广告上报。
- 可选的激进拦截：直播、视频、商城、活动入口。

### 2. Script 层

插件使用 `[Script]` 中的 `http-response` 规则，并设置 `requires-body=true` 读取 QQ 音乐 JSON 配置响应。脚本解析响应体，删除不想要的入口或广告对象，再返回清洗后的 body。

如果响应体不是 JSON，或 JSON 解析失败，脚本原样返回 body。

### 3. MITM 层

插件的 `[MITM]` hostnames 只覆盖 QQ 音乐相关域名。除非测试证明没有更窄的替代方案，否则不使用 `*.qq.com` 这类过宽规则。

初始候选域名：

- `y.qq.com`
- `u.y.qq.com`
- `c.y.qq.com`
- `i.y.qq.com`
- `musicpay.qq.com`
- `*.music.qq.com`

实现阶段可以根据 Loon 日志补充更窄的 QQ 音乐域名，但没有明确理由时不加入无关腾讯服务。

### 4. 清洗脚本

`qqmusic_cleaner.js` 提供通用递归 JSON 清洗逻辑，不绑定单个接口结构，因为 QQ 音乐响应格式可能变化。

脚本需要：

- 递归遍历对象和数组。
- 删除疑似广告、运营、弹窗、开屏、直播、短视频、商城、抽奖、装扮、活动的对象。
- 删除或置空明显承载广告/运营 payload 的字段。
- 保留疑似歌曲、专辑、歌手、歌单、歌词、播放数据、账号数据、支付/会员状态的对象。
- 如果清洗结果会破坏顶层结构，则回退为原始数据。

## 清洗规则

清洗器从字段名、展示文案和 URL 三个维度判断。

字段名或路径关键词：

- `ad`
- `ads`
- `advert`
- `advertisement`
- `splash`
- `popup`
- `pop`
- `banner`
- `operation`
- `activity`
- `h5`
- `live`
- `mall`
- `video`
- `shortvideo`
- `feed`
- `recommend`
- `trace`
- `report`

中文文案关键词：

- `广告`
- `开屏`
- `弹窗`
- `直播`
- `商城`
- `活动`
- `抽奖`
- `短视频`
- `看点`
- `扑通`
- `装扮`
- `福利`
- `运营`

URL 关键词：

- 广告和推广路径。
- H5 活动页。
- 曝光和点击上报接口。
- 直播、商城、短视频、运营活动落地页。

实现时不能只因为父级字段包含 `recommend` 就删除整块推荐内容。歌曲推荐列表属于核心体验，只有其中明确命中广告、运营、活动、直播、商城等特征的对象才删除。

## 安全行为

插件默认激进，但失败时必须温和回退：

- JSON 解析失败时返回原始 body。
- 非 JSON 响应返回原始 body。
- 清洗后顶层结果为空或结构异常时返回原始 body。
- 删除前先检查核心功能白名单。
- 规则按类别组织，方便用户只禁用某一类而不是移除整个插件。

## 测试计划

在 Loon 安装后：

1. 确认 MITM 证书已安装并受信任。
2. 启用插件。
3. 冷启动 QQ 音乐，检查开屏广告和启动弹窗是否减少。
4. 检查主 tab 和首页，确认直播、短视频、商城、活动、推广入口是否减少。
5. 测试登录状态、搜索、播放、歌词、歌单打开、专辑打开、收藏。
6. 如果目标功能异常，优先禁用最可能相关的 rewrite 分区，再禁用响应脚本。
7. 如果垃圾入口仍存在，查看 Loon 最近请求日志，补充更精确的 URL 模式或清洗关键词。

## 电脑操作安装路径

实现完成后，助手可以通过 Computer Use 操作 iPhone 镜像修改 Loon：

1. 在镜像 iPhone 中打开 Loon。
2. 导入生成的插件，或将插件内容粘贴到 Loon。
3. 确认已对列出的 hostnames 启用 MITM。
4. 如果 Loon 显示证书状态，确认 MITM 证书已信任。
5. 打开 QQ 音乐并执行测试计划。

用户已直接要求通过电脑操作 iPhone 镜像修改 Loon，因此修改本地 Loon 配置属于本任务范围。若后续步骤涉及传输敏感数据或高风险确认，助手应在执行前暂停并询问用户。
